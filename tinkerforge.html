Tinkerforge Component

The Tinkerforge Apache Camel Component for a simple access for use the Tinkerforge Daemon, Bricks and Bricklets.

Maven users will need to add the following dependency to their pom.xml for this component:

<dependency>
     <groupId>org.atomspace.camel</groupId>
     <artifactId>camel-tinkerforge</artifactId>
     <version>?.?.?</version>
     <!-- use the same version as your Camel core version -->
</dependency>

URI format

tinkerforge:[host[:port]/]deviceType?[uid=uid&options...]

URI Options

The options can be use for every Tinkerforge Component.

Name

Default Value

Description

host

localhost

Hostname / IP-Address

port

4223

Port

secret

null

Secret Token for authenticate

autoReconnect

true

reconnection on broken connection

timeout

2500

connection timeout

deviceType

Temperatur, Line, IO4, Color, MotionDetector, ...

uid

UID of the Device (Brick, Bricklet)

init

Configure the Device on Startup. This defined Functions will be called, the names have to seperate with Comma.

function

execute Device Function, can use for send CommandMessage to the ProducerEndpoint or use dynamic in Message Header for ProducerEndpoints Exchange. Every DeviceType have individual Functions.

callback

add Device Listener to ConsumerEndpoint for receive values, if callback is null all Listeners will be registrate. Every DeviceType have individual Listeners.
